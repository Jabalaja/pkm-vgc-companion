## Architecture Invariants Established

1. **Regulation-centric model:** Regulation is the top-level entity defining legal universe (Pokémon, items, gimmicks, Restricted count)
2. **Gimmick as list:** Multiple gimmicks stored as list, not single value (future-proof for stacked mechanics)
3. **English-only repo:** All code, comments, identifiers, docs, and commits in English
4. **Plan completeness rule:** Architecture/CI/build/schema changes must list affected docs and bundle updates in same PR
5. **Separation of concerns:** Local AI config gitignored; repo-scoped config tracked; cloud executor config tracked

- **Plan handoff:** GitHub Issues with `plan` template; lifecycle labels: `plan` → `plan:ready` → `plan:in-progress` → closed
- **Optional deep specs:** `docs/plans/NNN-slug.md` for architectural or multi-step migrations
- **Automated label management:** Workflows auto-flip on assignment and clean on closure
- **Auto-merge for Copilot:** `copilot/*` branches auto-merge after approval, CI green, and sync with main
- **Cloud executor config:** `.github/copilot-instructions.md` mirrors local planning guide; read by Copilot on session start
