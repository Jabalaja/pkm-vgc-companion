## Key Decisions Made

1. **PokeAPI as Pokémon data source:** Selected over alternatives because it offers comprehensive, well-maintained coverage of species, forms, moves, items, abilities, and stat data — exactly what a competitive team builder needs.

2. **Three-issue decomposition for Phase-2:** Sequential work that respects system boundaries:
   - Design tokens first (CSS, theming foundation).
   - Backend seeding + mutations (Convex data layer).
   - Frontend search + integration (reactive UI).

3. **Plan-completeness rule:** Prevents future regressions where architecture or CI changes land without corresponding documentation updates. Centralizes ownership of spec, code, and config in a single PR per issue.

4. **Executor assignment workflow:** Automated label flip (`plan:ready` → `plan:in-progress`) on Copilot assignment reduces ceremony and manual tracking overhead.
