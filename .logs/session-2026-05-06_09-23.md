## Key Decisions Made

1. **Deploy Key Strategy** — Chose a Preview Deploy Key (not Production) to minimize blast radius if credentials are leaked. This is safe because the preview deployment is ephemeral and per-developer; it cannot break production data or deployments.

2. **Validation Placement in CI** — Placed the Convex schema validation step between Lint and Build. This ensures format checks pass before attempting a schema push, avoiding false negatives due to formatting issues.

3. **Deferral of Other CI Items** — E2E, visual regression, and Lighthouse PWA scoring were documented in README as Phase 2+ work. These are valuable but add significant complexity and maintenance burden; Phase 1 is deliberately narrow.

4. **Discarding Auto-Rebase** — Auto-rebase conflicts with GitHub's Main ruleset and auto-merge flow. The existing `require_last_push_approval=true` with auto-update-branch exemptions is sufficient; adding auto-rebase would over-complicate the workflow.

## Problems Encountered and Solutions

### Problem 1: Production vs. Preview Deploy Key
**Symptom:** Initial recommendation used a Production Deploy Key.
**Root cause:** Did not adequately reason about the blast radius of credential leakage.
**Solution:** After Chris's push-back, switched to Preview Deploy Key strategy. Preview deployments are ephemeral and cannot compromise production.
**Lesson:** For vendors with ephemeral environments, always default to the non-production credential tier.

### Problem 2: Convex Codegen Offline Assumption
**Symptom:** Plan assumed `convex codegen` could run without network access.
**Root cause:** Misunderstood Convex tooling capabilities; many codegen tools run offline, but Convex requires deployment context.
**Solution:** Pivoted to deploy-key strategy, which pushes schema changes to an ephemeral preview and relies on Convex's own validation.
**Lesson:** Always run local smoke tests on tooling assumptions before committing to a plan.

### Problem 3: Biome Format Failure on First Negative Test Push
**Symptom:** PR #7 failed on `pnpm check` (Biome formatting) rather than reaching the Convex validation step.
**Root cause:** Did not run `pnpm check` locally before pushing the deliberate schema error.
**Solution:** Ran `pnpm format` locally and re-pushed. Second push correctly reached the Convex validation step and failed as designed.
**Lesson:** Always run `pnpm check` before pushing to catch linting and format issues upstream.

## Code Changes Summary

### `.github/workflows/ci.yml`
Added new step between Lint and Build:

```yaml
- name: Convex schema validation (push to ephemeral preview)
  run: npx convex push --yes
  env:
    CONVEX_DEPLOY_KEY: ${{ secrets.CONVEX_DEPLOY_KEY }}
```

This step ensures that any schema changes compile correctly and do not break existing queries or mutations. The Preview Deploy Key is used so that even if credentials leak, only an ephemeral, per-developer deployment is affected.
